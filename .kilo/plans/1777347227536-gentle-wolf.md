# PDF Explain — Robustness & Edge Case Fixes Plan

## Overview
Comprehensive fix plan addressing all identified issues: context invalidation, button placement, extraction reliability, error handling, memory leaks, race conditions, and browser compatibility.

---

## Issue 1: Extension Context Invalidation (Critical)

**Problem**: Content script destroyed during async operations (streaming), causing "Extension context invalidated" errors.

### Fixes:

#### 1.1 Add chrome.runtime.connect for Background Worker
- Modify `background.js` to handle long-lived operations
- Content script delegates streaming to background worker via message passing
- Background worker maintains state even if content script is destroyed

**Files**: `background.js`, `content.js`

#### 1.2 Enhanced Cleanup Handlers
- Add `visibilitychange` handler (page hidden/visible)
- Add `pagehide` handler (more reliable than unload)
- Store cleanup functions in a Set for batch removal
- Add try-catch wrapper around all async operations

**Files**: `content.js`

#### 1.3 Visibility-Aware Streaming
- Pause streaming when page becomes hidden
- Resume when page becomes visible (or cancel if too long)
- Check `document.hidden` before DOM updates

**Files**: `content.js`

## Issue 2: Button Placement Issues

**Problem**: Toggle button overlaps toolbar in Brave/Chromium. Fixed position conflicts with viewer.

### Fixes:

#### 2.1 Expanded Toolbar Selectors
Add more common PDF viewer toolbar selectors:
```javascript
const toolbarSelectors = [
  '#toolbar', '.toolbar', '.viewerToolbar', '.pdf-toolbar',
  '[role="toolbar"]', '.pdfViewer .toolbar', '#pdf-viewer-toolbar',
  '.chrome-pdf-toolbar', 'embed[src$=".pdf"] + *', // Common patterns
  '.viewer's-side-bar', // Some viewers
];
```

#### 2.2 MutationObserver for Dynamic Toolbar
- Watch for toolbar appearing after page load
- Move button if toolbar detected later

#### 2.3 Multiple Position Strategies
1. Try to append to toolbar first
2. If toolbar has a dedicated button container, use it
3. Fallback: Fixed position with bottom alignment (below toolbar)
4. Last resort: Floating draggable button

**Files**: `content.js`, `styles/main.css`

## Issue 3: Text Extraction Reliability

**Problem**: Extraction fails on some PDFs (CORS, scanned images, complex layouts).

### Fixes:

#### 3.1 Blob URL Memory Management
- Store blob URLs in a Set
- Revoke all blob URLs when switching to PDF mode
- Revoke on cleanup

#### 3.2 Enhanced Page Number Detection
- Check URL hash `page=N`
- Check viewer.currentPageNumber
- Check `window.pdfjsViewer.currentPageNumber`
- Check `.page[data-page-number]` attribute
- Check `input[type="number"][value]` in toolbar
- Default to 1, never return undefined

#### 3.3 PDF.js Loading Robustness
- Add timeout for worker initialization
- Catch and handle worker creation errors
- Fallback to DOM extraction if PDF.js fails completely
- Add retry logic (3 attempts with backoff)

#### 3.4 CORS Handling
- Try with credentials: 'include'
- Try without credentials (some CDNs block cookies)
- Try different origin handling
- Clear error message if all fail

#### 3.5 Scanned/Image Detection
- If text extraction returns empty/very short, show specific message
- Detect if PDF has embedded images
- Provide guidance: "This appears to be a scanned document"

**Files**: `content.js`

## Issue 4: Error Handling & User Experience

**Problem**: Users see cryptic errors, no retry options, no degradation.

### Fixes:

#### 4.1 Structured Error Messages
Create error categories:
- `NETWORK_ERROR`: "Unable to connect. Check your internet connection."
- `EXTRACTION_ERROR`: "Could not read this PDF. Try a different file."
- `LLM_ERROR`: "AI service temporarily unavailable. Please try again."
- `RATE_LIMIT`: "Too many requests. Please wait a few minutes."

#### 4.2 Retry with Exponential Backoff
- First retry: immediate
- Second retry: 1 second delay
- Third retry: 2 seconds delay
- After 3 failures: show permanent error with manual retry button

#### 4.3 Graceful Degradation
- If LLM fails: show extracted text (original, not rewritten)
- If extraction fails: show helpful message with suggestions
- Cache working results for offline viewing

#### 4.4 Timeout Handling
- Network timeout: 15 seconds
- Extraction timeout: 30 seconds
- Streaming timeout: 60 seconds (with progress indicator)

**Files**: `content.js`

## Issue 5: Race Conditions & State Management

**Problem**: Multiple rapid clicks cause concurrent operations, corrupted state.

### Fixes:

#### 5.1 Operation Queue
- Queue operations instead of running concurrently
- Cancel pending operations before starting new ones
- Use a single AbortController for entire pipeline

#### 5.2 State Flags
- Add `isInitializing` flag
- Add `isExtracting` flag
- Add `isStreaming` flag
- Check flags before starting new operations

#### 5.3 Button State Management
- Disable button during operations
- Show loading spinner/state
- Prevent double-clicks

**Files**: `content.js`

---

## Issue 6: Memory Leaks

**Problem**: Unbounded cache growth, unreleased blob URLs, lingering listeners.

### Fixes:

#### 6.1 Cache Limits
- Maximum cache size: 50 entries
- Use LRU (Least Recently Used) eviction
- Clear cache when URL changes (new PDF)

#### 6.2 Blob URL Cleanup
- Track all created blob URLs
- Revoke on: mode switch, cleanup, page unload
- Use WeakRef for tracking (if available)

#### 6.3 Event Listener Cleanup
- Remove all event listeners in cleanup()
- Use AbortSignal for fetch operations
- Clear all timeouts/intervals

**Files**: `content.js`, `background.js`

---

## Issue 7: Browser Compatibility

**Problem**: Different browsers and PDF viewers have different structures.

### Fixes:

#### 7.1 Extended Feature Detection
- Detect Chrome native viewer
- Detect Firefox PDF.js viewer
- Detect Microsoft Edge viewer
- Detect third-party viewers (PDF.js standalone, etc.)

#### 7.2 Viewer-Specific Selectors
Build selector map for different viewers:
```javascript
const viewerConfigs = {
  chrome: { container: '#viewer', page: '.page', toolbar: '#toolbar' },
  pdfjs: { container: '#pdf-viewer', page: '.page', toolbar: '.toolbar' },
  edge: { container: '#pdfViewer', page: '[data-page-number]', toolbar: '.chrome-pdf-toolbar' },
};
```

#### 7.3 Feature Detection Over UA String
- Use feature detection instead of UA sniffing
- Test for specific APIs/elements

**Files**: `content.js`

---

## Issue 8: Streaming Improvements

**Problem**: AbortController doesn't fully cancel, no progress feedback, no resume.

### Fixes:

#### 8.1 Proper Stream Cancellation
- Use `reader.cancel()` in addition to AbortController
- Handle `AbortError` specifically for streaming
- Ensure reader.releaseLock() is always called

#### 8.2 Progress Indicator
- Show token count as streaming progresses
- Show estimated time remaining (based on tokens/sec)
- Allow cancellation mid-stream

#### 8.3 Chunk Buffering
- Reduce flush frequency from 50ms to 100ms for performance
- Accumulate larger chunks before DOM update
- Use `requestAnimationFrame` for smooth updates

**Files**: `content.js`
